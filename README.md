## 游戏玩法

1. **观察宝石图片**，点击词条池中的词条添加至倍率区（最多 8 个）
2. **倍率区内触摸拖拽排序**，结算时按顺序逐一翻牌
3. **调整筹码** — ×2 翻倍 / ÷2 减半 / ALL IN 全押（每局限一次）
4. **开牌** — 词条逐一翻转揭示倍率，乘积结算
5. **正倍率加分**，**零倍率 Bust 归零**，**负倍率立即停牌扣分**
6. 共 **10 题**，筹码归零或完成全部题目后结算最终排名

## 技术栈

| 层 | 选型 |
|----|------|
| HTML | 纯 HTML5，四屏状态机 |
| CSS | 自定义属性体系 · `Noto Sans SC` + `Space Mono` 字体 · CRT 扫描线覆层 |
| JS | 原生 ES6 · 事件委托 · `getBoundingClientRect` 碰撞检测 · CSS transition 飞行动画 |
| 音效 | ZzFX 内联引擎 · 程序化合成（三角波/方波/锯齿波） · AudioContext 自动恢复 · 移动端手势解锁 |
| 存储 | `localStorage`（排行榜，最多 100 条） |
| 数据 | `questions.json` — 非开发人员可直接编辑扩展题库 |

**零依赖、零构建** — 打开即玩，部署只需上传静态文件。

## 扩展指南

- **加题**：编辑 `questions.json`，每条包含 `id / image / gemName / terms[]`（详见已有条目格式）
- **加图**：放入 `images/` 目录，文件名在题库中引用
- **改规则**：`script.js` 顶部变量控制回合数（`currentRound > 10`）、初始筹码、彩蛋概率等
- **改样式**：`styles.css` 顶部 `:root` 自定义属性控制全套配色

### 2026-06-20 — 音效系统 & 交互打磨

- **ZzFX 音效引擎**内联集成（MIT 许可），8 种程序化音效：金币碰撞 (×2/ALL IN)、词条选择/移除、翻牌正确/错误/负倍率
